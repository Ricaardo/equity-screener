# A 股账户可交易的海外、商品与资源 ETF/LOF 清单

数据口径：本地 `data/ah_screener.duckdb` 的 `market_snapshots` 最新 A 股 ETF/LOF 快照，交易日 `2026-05-26`；成交额单位为亿元。下表用于研究和筛选，不构成投资建议。

## 结论

A 股账户可以间接投资海外资产，主要路径是场内 QDII ETF、跨境 ETF、跨境 LOF、港股通主题 ETF、商品期货/实物类 ETF/LOF。交易在境内交易所完成，通常以人民币计价，不等于直接持有海外个股。

需要分清三类工具：

- 跨境 ETF/LOF：投向香港、美国、日本、德国、法国、印度、巴西、沙特、东南亚等市场；适合做海外 beta 或海外行业主题暴露。
- 商品/期货/实物类：黄金、白银、豆粕、原油、能源化工、大宗商品等；底层多为实物黄金、商品期货或 QDII 商品相关工具。
- 资源权益类 ETF：有色、黄金股、油气/石油股等，底层是股票，不是商品期货；交易和风险更接近行业股票 ETF。

跨境 ETF/LOF、债券 ETF、黄金 ETF、货币 ETF 等场内基金存在 T+0 回转交易制度安排，但跨境产品有节假日错配、汇率、QDII 额度、净值滞后、二级市场溢价/折价等风险。

## 筛选规则

1. 先获取全量 A 股场内 ETF/LOF，再筛选海外、商品、资源权益等方向，不从少数手工代码开始。
2. 先按底层资产分组，不按名字分组。同样叫“油气/石油”的产品可能是海外油气股票、原油/石油 LOF，也可能是境内油气行业 ETF。
3. 同组优先看标的分布占比；如果有披露持仓，按前十大持仓权重相似度去重；如果没有持仓但有行业/资产配置，按配置权重去重。
4. 持仓数据缺失时，再保守回退到“类别 + 细分 track”去重，避免把主动海外 LOF 仅凭名字粗暴合并。
5. 同组优先看成交额和价差。日成交额低于 `2000 万` 的标的只做备查；`1 亿` 以上才适合进入优先观察池。
6. LOF/QDII 必须额外看溢价率、申赎状态、QDII 额度和最新净值日期。净值滞后会让盘中价格与真实资产偏离。
7. 主动/半主动 LOF 适合表达“基金经理可调整持仓”的想法，但要接受风格漂移、费率、持仓披露滞后和经理能力风险。
8. 被动 ETF 更适合精确表达 beta，例如纳指、标普、恒生科技、日经；主动 LOF 更适合主题精选，例如港美互联网、海外科技。

## 全量筛选与去重流程

当前代码已支持三步：

```bash
uv run ah-screener sync-spot --market ETF
uv run ah-screener sync-etf-exposures --market A --year 2026
uv run ah-screener etf-export --market A --grouped --dedup-by exposure --top 100
```

输出里的关键字段：

- `etf_category`：宽基、跨境、商品、行业、债券、货币等大类。
- `etf_track`：恒生科技、纳斯达克100、标普500、白银、豆粕、黄金股、油气股等细分方向。
- `etf_dedup_basis`：`holding_seed/holding_overlap` 表示按持仓权重去重；`allocation_seed/allocation_overlap` 表示按行业或资产配置去重；`rule_seed/rule_fallback` 表示数据缺失时按类别和 track 保守去重。
- `etf_top_holdings`：已披露的主要持仓及权重。
- `etf_primary_allocation`：已披露的主要行业/资产配置。
- `peer_alternatives`：同组被折叠的可替代标的。

商品实物、期货、QDII 商品 LOF 经常没有股票持仓披露，所以会更多依赖类别和 track 兜底。海外主动 LOF 有持仓时会按持仓相似度判断；没有持仓时默认保持更分散，等待人工核验。

## 当前优先观察框架

- 海外主动/可调仓：`160644`、`501312`、`501225`。先核验持仓、基金经理、费率和溢价。
- 海外被动 beta：恒生科技、港股互联网、纳指/纳斯达克100、标普500、日经225。先按成交额和折溢价选一只。
- 商品期货/实物：黄金、白银、豆粕、原油。先看标的机制，再看成交额。
- 资源权益：有色金属、黄金股、油气/石油股。按行业 ETF 处理，不按商品 ETF 处理。

## 可复现命令

```bash
uv run ah-screener sync-spot --market ETF
uv run ah-screener etf-export --market A --raw --top 300
```

也可以只看去重后的工具池：

```bash
uv run ah-screener etf-export --market A --top 80
```

如果要按标的分布占比去重，先同步披露持仓和行业/资产配置：

```bash
uv run ah-screener sync-etf-exposures --market A --year 2026
uv run ah-screener etf-export --market A --grouped --dedup-by exposure --top 80
```
